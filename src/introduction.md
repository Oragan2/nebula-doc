# Introduction

The Nebula language is made for people searching to write a kernel or low level application and not wanting to have C or C++ issues with inline assembly, memory safety and more

## Chapters: 

This documentation is divided into a few chapters : 
- [Memory safety & layout](memory-safety.md) -> These chapters will cover how the memory is stored for variables and the safety put in place
- [Types](types.md) -> These chapters will cover the types present within the language from the classic types to the more specialized ones
- [Memory Access](memory-access.md) -> These chapters will cover how you can access memory within the language
- [Data Structures](data-structures.md) -> These chapters will cover how you can create data structures to store data in a way
- [Inline assembly](inline-assembly.md) -> This chapter will cover how inline assembly is written
